# MVP Question Rules

Use these rules while creating the first MVP.

The goal is to launch quickly and learn from real usage.

## Question Rules

Ask only questions that affect:

- MVP scope
- first user
- first payment
- core user value
- launch readiness
- analytics/tracking
- feedback collection
- major technical blockers
- decisions that would make the MVP hard to upgrade later

Avoid questions about:

- enterprise scale
- advanced permissions
- large-team workflows
- complex admin systems
- advanced security beyond basic safety
- perfect architecture
- future features not needed for validation
- optimization before real usage

## Final Product Concerns

Mention final-product concerns only if ignoring them now would create expensive rework later.

Examples:

- Auth/user structure: important now
- Database ownership/user IDs: important now
- Payment tracking: important now
- Analytics events: important now
- Enterprise roles/teams: not important now
- Advanced dashboards: not important now
- Complex automation: not important now

## Output Style

Ask one question at a time.

For each question:

1. Explain why it matters for the MVP.
2. Give your recommended answer.
3. State whether this is:
   - Must decide before build
   - Can decide later
   - Not needed for MVP

Do not start coding.

Do not create a full enterprise design.

Keep pushing toward a small launchable MVP.
