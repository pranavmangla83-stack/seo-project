# Build Rules

1. Build vertical slices only.

2. No issue bigger than 90 minutes.

3. Every issue must be testable.

4. Every issue must move toward:
   - first user
   - first payment
   - first feedback

5. No feature without validation.

6. Prefer existing libraries over custom code.

7. Launch before perfect.

8. Fix root cause, not symptoms.

9. Finish current issue before starting another.

10. Read AI_CONTEXT.md before coding.

11. Read current GitHub Issue before coding.

12. Do not over-engineer MVP.

13. If unsure, choose the simpler solution.

14. Reuse existing code/templates whenever possible.

15. Every feature must answer:
    "How does this help get users, payments, or learning?"
